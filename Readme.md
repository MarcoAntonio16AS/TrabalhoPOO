                                --- DESCRIÇÃO DO TRABALHO ---
                        Projetos em Grupo (POO + Java Console + MySQL)
                    _______________________________________________________




> **Objetivo da atividade**
------------------------------


Nesta etapa da disciplina, cada grupo irá desenvolver um mini-sistema de software em **Java** (console) inspirado em um cenário real do mercado (Help Desk, Logística, Estoque, Agenda de Clínica, SaaS, Reservas, Frotas, EdTech, Fintech/ERP, Locação, Eventos, RH, Kanban, Biblioteca e Procurement). O propósito é transformar conteúdos vistos em sala em um projeto funcional, com regras de negócio claras, menu interativo e relatórios — do mesmo jeito que acontece em sistemas corporativos. Para fins de conextualização o Projeto deverá ter acesso a um banco de dados Mysql tornando o projeto completo e em linha com demais disciplinas já vistas.






> **O que vocês vão aprender durante o projeto?**
------------------------------------------------------


Ao implementar o sistema, o grupo vai praticar (na prática e com código real):


1) **Modelagem orientada a objetos**: transformar um problema do mundo real em classes, atributos e métodos com responsabilidade bem definida.

2) **Encapsulamento e validações**: proteger os dados do sistema e garantir consistência (evitar valores inválidos e “estado errado”).

3) **Herança e generalização/especificação**: criar uma classe base e especializações (por exemplo, Chamado → Incidente/Requisicao, Entrega → Expressa/Agendada, Produto → Perecível/Eletrônico), reduzindo repetição e organizando o domínio.

4) **Sobreposição de métodos (override) e polimorfismo**: implementar regras diferentes para cada tipo e, ao mesmo tempo, usar uma referência mais geral (ex.: List <Chamado> com subclasses) para que o sistema execute automaticamente o comportamento correto.

5) **Estruturas de decisão e repetição**: construir um menu robusto e fluxos que simulam a operação do sistema (cadastro, atendimento, validação, cancelamento, etc.).

6) **Coleções e coleções de objetos**: armazenar dados em listas e mapas, além de usar atributos que são coleções (ex.: histórico de comentários, eventos, movimentações, itens de pedido).

7) **Relatórios e visão gerencial**: gerar saídas que ajudam a tomar decisões (prioridades, totais, ranking, atrasos, faturamento, estoque baixo, etc.).






> **O que vocês terão aprendido ao final?**
----------------------------------------------


Ao concluir o projeto, vocês terão produzido um sistema completo e executável, e principalmente terão desenvolvido competências que aparecem diretamente em entrevistas e no dia a dia profissional:



**Pensar como analista/desenvolvedor, traduzindo requisitos em implementação;**
**Projetar classes e regras de negócio com clareza;**
**Organizar um projeto em pacotes (model/service/ui) e manter o código legível e comentado;**
**Testar e demonstrar o software com segurança, explicando decisões e resultados;**
**Criar um trabalho que pode ser evoluído futuramente para GUI, API, banco de dados e deploy.**






> **Mensagem final!**
------------------------

Encarem este projeto como o primeiro “produto” do seu portfólio: ele não é apenas um exercício, é uma oportunidade de demonstrar que vocês conseguem criar um sistema do zero, com estrutura, regras e entrega.


Dividam responsabilidades no grupo, façam pequenas entregas semanais, testem o menu com frequência e usem o tema do seu grupo para construir algo que dê orgulho de apresentar. No final, cada grupo terá não só um sistema pronto, mas também a experiência real de planejar, implementar e validar um software orientado a objetos.
